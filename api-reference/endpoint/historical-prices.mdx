---
title: 'Precios Históricos'
openapi: 'GET /api/v1/market_data/historicals/prices/{symbol}'
---

<Note>
  Este endpoint obtiene datos de precios históricos para un instrumento financiero específico durante un rango de fechas determinado.
</Note>

## Parámetros de Ruta

<ParamField path="symbol" type="string" required>
  El símbolo ticker o identificador del instrumento financiero (ej., "PAGS", "GGAL", "YPFD").
</ParamField>

## Parámetros de Consulta

<ParamField query="from_date" type="string" required>
  Fecha de inicio para los datos históricos en formato YYYY-MM-DD.
</ParamField>

<ParamField query="to_date" type="string" required>
  Fecha de fin para los datos históricos en formato YYYY-MM-DD.
</ParamField>

<ParamField query="adjusted" type="boolean" required>
  Si devolver precios ajustados por eventos corporativos (splits, dividendos, etc.). Establecer en `true` para precios ajustados, `false` para precios sin ajustar.
</ParamField>

## Ejemplo de Solicitud

```bash
curl -X GET "https://api.doctacapital.com.ar/api/v1/market_data/historicals/prices/PAGS?from_date=2025-06-01&to_date=2025-06-15&adjusted=false" \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
```

## Respuesta Exitosa

<ResponseField name="symbol" type="string">
  El símbolo ticker del instrumento solicitado.
</ResponseField>

<ResponseField name="data" type="array">
  Array de puntos de datos de precios históricos.
</ResponseField>

<ResponseField name="data[].date" type="string">
  La fecha de los datos de precio en formato YYYY-MM-DD.
</ResponseField>

<ResponseField name="data[].open" type="number">
  Precio de apertura del día.
</ResponseField>

<ResponseField name="data[].high" type="number">
  Precio más alto durante el día.
</ResponseField>

<ResponseField name="data[].low" type="number">
  Precio más bajo durante el día.
</ResponseField>

<ResponseField name="data[].close" type="number">
  Precio de cierre del día.
</ResponseField>

<ResponseField name="data[].volume" type="integer">
  Volumen de negociación del día.
</ResponseField>

<ResponseField name="adjusted" type="boolean">
  Si los precios están ajustados por eventos corporativos.
</ResponseField>

<ResponseField name="from_date" type="string">
  La fecha de inicio del rango de datos solicitado.
</ResponseField>

<ResponseField name="to_date" type="string">
  La fecha de fin del rango de datos solicitado.
</ResponseField>

## Respuestas de Error

### 400 Solicitud Incorrecta
Formato de fecha inválido o parámetros requeridos faltantes.

```json
{
  "type": "/errors/bad-request",
  "title": "Parámetros de solicitud inválidos",
  "status": 400,
  "detail": "Formato de fecha inválido. Usar formato YYYY-MM-DD.",
  "correlation_id": "12345678-1234-1234-1234-123456789012"
}
```

### 401 No Autorizado
Token de acceso inválido o faltante.

```json
{
  "type": "/errors/authentication-required",
  "title": "Token de acceso inválido o faltante",
  "status": 401,
  "detail": "Se requiere token de acceso para este endpoint",
  "correlation_id": "12345678-1234-1234-1234-123456789012"
}
```

### 404 No Encontrado
Símbolo no encontrado o no hay datos disponibles para el rango de fechas especificado.

```json
{
  "type": "/errors/not-found",
  "title": "Símbolo no encontrado",
  "status": 404,
  "detail": "No hay datos disponibles para el símbolo 'INVALID' en el rango de fechas especificado",
  "correlation_id": "12345678-1234-1234-1234-123456789012"
}
```

## Notas

- El rango de fechas no puede exceder 1 año
- Máximo 1000 puntos de datos por solicitud
- Los datos están disponibles solo para días hábiles (excluyendo fines de semana y feriados)
- Los precios están en la moneda local del instrumento

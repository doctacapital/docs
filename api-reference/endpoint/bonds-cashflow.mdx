---
title: 'Flujo de Caja de Bonos'
openapi: 'GET /api/v1/market_data/bonds/{symbol}/cashflow'
---

<Note>
  Este endpoint obtiene el cronograma de flujo de caja para un bono específico, mostrando todos los pagos futuros incluyendo capital e intereses.
</Note>

## Parámetros de Ruta

<ParamField path="symbol" type="string" required>
  El símbolo del bono (ej., "AL30", "GD30", "AE38").
</ParamField>

## Parámetros de Consulta

<ParamField query="nominal_units" type="integer" required>
  Unidades nominales para el cálculo. Esto determina el valor nominal de la posición del bono para la cual calcular el flujo de caja.
</ParamField>

## Ejemplo de Solicitud

```bash
curl -X GET "https://api.doctacapital.com.ar/api/v1/market_data/bonds/AL30/cashflow?nominal_units=100" \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
```

## Respuesta Exitosa

<ResponseField name="symbol" type="string">
  El símbolo del bono solicitado.
</ResponseField>

<ResponseField name="nominal_units" type="integer">
  Las unidades nominales utilizadas para el cálculo.
</ResponseField>

<ResponseField name="cashflow" type="array">
  Array de eventos de flujo de caja para el bono.
</ResponseField>

<ResponseField name="cashflow[].date" type="string">
  La fecha de pago en formato YYYY-MM-DD.
</ResponseField>

<ResponseField name="cashflow[].type" type="string">
  El tipo de pago (ej., "interest", "principal", "amortization").
</ResponseField>

<ResponseField name="cashflow[].amount" type="number">
  El monto del pago en la moneda del bono.
</ResponseField>

<ResponseField name="cashflow[].currency" type="string">
  La moneda del pago (ej., "ARS", "USD").
</ResponseField>

<ResponseField name="cashflow[].description" type="string">
  Descripción del pago (ej., "Pago de Intereses", "Amortización de Capital").
</ResponseField>

<ResponseField name="total_interest" type="number">
  Total de pagos de intereses durante la vida del bono.
</ResponseField>

<ResponseField name="total_principal" type="number">
  Total de pagos de capital durante la vida del bono.
</ResponseField>

<ResponseField name="total_amount" type="number">
  Monto total del flujo de caja durante la vida del bono.
</ResponseField>

## Respuestas de Error

### 400 Solicitud Incorrecta
Parámetros inválidos o campos requeridos faltantes.

```json
{
  "type": "/errors/bad-request",
  "title": "Parámetros de solicitud inválidos",
  "status": 400,
  "detail": "nominal_units debe ser un entero positivo",
  "correlation_id": "12345678-1234-1234-1234-123456789012"
}
```

### 401 No Autorizado
Token de acceso inválido o faltante.

```json
{
  "type": "/errors/authentication-required",
  "title": "Token de acceso inválido o faltante",
  "status": 401,
  "detail": "Se requiere token de acceso para este endpoint",
  "correlation_id": "12345678-1234-1234-1234-123456789012"
}
```

### 404 No Encontrado
Bono no encontrado o no hay datos de flujo de caja disponibles.

```json
{
  "type": "/errors/not-found",
  "title": "Bono no encontrado",
  "status": 404,
  "detail": "No hay datos de flujo de caja disponibles para el símbolo 'INVALID'",
  "correlation_id": "12345678-1234-1234-1234-123456789012"
}
```

## Notas

- Los datos de flujo de caja incluyen todos los pagos futuros hasta el vencimiento del bono
- Los montos se calculan basándose en las unidades nominales especificadas
- Las fechas de pago son en días hábiles (los fines de semana y feriados pueden afectar las fechas reales de pago)
- Los pagos de intereses son típicamente semestrales para los bonos argentinos
- Los pagos de capital pueden incluir amortizaciones parciales o pagos bullet al vencimiento
